# Release Process

This document describes how to build and publish a new release of **Property Management LeadGen**.

---

## Prerequisites

| Tool | Version | Install |
|------|---------|---------|
| Rust | stable | `rustup install stable` |
| Node.js | 20+ | `nvm install 20` |
| Python | 3.13 | `uv python install 3.13` |
| Tauri CLI | 2.x | `cargo install tauri-cli` |
| PyInstaller | 6+ | `uv pip install pyinstaller` |

## Before You Release

### 1. Generate Updater Signing Keys (one-time)

Skip this if you already have keys from the initial setup:

```bash
# Using Tauri CLI (recommended)
cargo tauri signer generate -w ~/.tauri/leadgen.key

# Or using Python (if no Tauri CLI available)
python3 -c "
import base64, os
from cryptography.hazmat.primitives.asymmetric import ed25519
from cryptography.hazmat.primitives import serialization
os.makedirs(os.path.expanduser('~/.tauri'), exist_ok=True)
key = ed25519.Ed25519PrivateKey.generate()
with open(os.path.expanduser('~/.tauri/leadgen.key'), 'wb') as f:
    f.write(key.private_bytes(
        encoding=serialization.Encoding.Raw,
        format=serialization.PrivateFormat.Raw,
        encryption_algorithm=serialization.NoEncryption(),
    ))
os.chmod(os.path.expanduser('~/.tauri/leadgen.key'), 0o600)
pub = key.public_key().public_bytes(
    encoding=serialization.Encoding.Raw,
    format=serialization.PublicFormat.Raw,
)
print(f'Public key: {base64.b64encode(pub).decode()}')
```

Copy the **public key** into `src-tauri/tauri.conf.json` → `plugins.updater.pubkey`.

### 2. Set GitHub Secrets

Add these secrets to the repository at `Settings → Secrets and variables → Actions`:

| Secret | Value |
|--------|-------|
| `TAURI_SIGNING_PRIVATE_KEY` | Contents of `~/.tauri/leadgen.key` |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | (empty or a passphrase if you set one) |

For macOS notarization (optional but recommended):

| Secret | Value |
|--------|-------|
| `APPLE_ID` | Your Apple Developer account email |
| `APPLE_PASSWORD` | App-specific password |
| `APPLE_TEAM_ID` | Your Apple Team ID |

### 3. Update the Version

Update the version in:

1. **`pyproject.toml`** — `version = "X.Y.Z"`
2. **`src-tauri/Cargo.toml`** — `version = "X.Y.Z"`
3. **`src-tauri/tauri.conf.json`** — `"version": "X.Y.Z"`

### 4. Verify Everything

```bash
# Python tests + lint
cd /home/turin/property-management-leadgen
uv run pytest tests/ -q --tb=no
ruff check src/ tests/

# Frontend build
cd frontend && npm ci && npm run build && cd ..

# Rust check
cargo check --manifest-path src-tauri/Cargo.toml

# Build the sidecar binary
uv run pyinstaller --clean --onedir --name python-sidecar \
  --collect-all src --hidden-import usaddress runner.py

# Quick smoke test
echo '{"method":"ping"}' | timeout 10 ./dist/python-sidecar/python-sidecar
```

## Publishing a Release

### Automated (via GitHub Actions)

The release pipeline in `.github/workflows/release.yml` handles everything:

1. **Tag the release:**
   ```bash
   git tag vX.Y.Z
   git push origin vX.Y.Z
   ```

2. **The CI does the rest:**
   - Builds the Python sidecar with PyInstaller
   - Builds the frontend with `npm ci && npm run build`
   - Runs `cargo tauri build` for All three platforms
   - Uploads installers as GitHub Release assets
   - Generates release notes

3. **Download** the installers from the GitHub Release page.

### Manual (for testing)

```bash
# Build Tauri app (output in src-tauri/target/release/bundle/)
cargo tauri build --manifest-path src-tauri/Cargo.toml

# The installer will be at:
# Linux:   src-tauri/target/release/bundle/deb/leadgen_X.Y.Z_amd64.deb
# macOS:   src-tauri/target/release/bundle/dmg/LeadGen_X.Y.Z_aarch64.dmg
# Windows: src-tauri/target/release/bundle/msi/LeadGen_X.Y.Z_x64.msi
```

## How Users Get Updates

Once the app is installed:

1. User opens the app
2. The app checks `https://github.com/iknowkungfubar/property-management-leadgen/releases/latest/download/updater.json`
3. If a newer version is found, the user is prompted with a dialog:
   > "A new version of LeadGen is available. Update now?"
4. On acceptance, the new version is downloaded and installed automatically
5. The app restarts with the new version

The `updater.json` is auto-generated by the release CI pipeline.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `TAURI_SIGNING_PRIVATE_KEY not set` | Add the private key to GitHub Secrets |
| `pubkey is empty` | Generate a keypair and update `tauri.conf.json` |
| Build fails on macOS | Install Xcode CLI tools: `xcode-select --install` |
| Build fails on Windows | Install Visual Studio Build Tools + Windows SDK |
| Sidecar binary missing | Run `pyinstaller` step before `cargo tauri build` |
| Code signing error | Check Apple Developer account is active |
